User: a person who is using a registered client to access a resource

Relying party (RP):an app/website that outsources its user authentication function to an IDP (Identity provider)

User Agent: a piece of s/w that requests the tokens either for authenticating an user or for accessing a resource

Open ID Provider (OP) or IDP (Identity provider): another entity that has implemented the Open ID Connect and OAuth2 protocols

Authentication: the secure process to establish and communicate between the person/user operating an app or browser is who they claim to be

1. End user navigates to a website or web app via a browser
2. end user clicks sign-in and enters the username and password
3. the client (RP) sends a request to IDP or OP [client must be registered with OP/IDP]
4. the IDP/OP authenticates the user and obtains authorization
5. The OP/IDP responds with and identity token and an access token
6. the RP can send a request with the access token to the user device
7. the "UserInfo" endpoint returns claims about an end user
